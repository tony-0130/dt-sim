# dt-sim

**A comprehensive Device Tree simulation toolchain that provides transparent, human-readable compilation and overlay merging.**

## 🎯 Why dt-sim?

When working with Device Tree overlays, standard tools like `dtc` and `fdtoverlay` often fail with cryptic error messages like "failed to apply overlay" without telling you what went wrong or where. dt-sim solves this by:

- ✅ **Transparent compilation** - See exactly what happens at each step
- ✅ **Detailed error reporting** - Precise error locations with helpful suggestions  
- ✅ **Human-readable output** - Text-based DTB files instead of binary blobs
- ✅ **Complete validation** - Comprehensive testing of references and overlays
- ✅ **Production-ready** - Handles complex SoC device trees like RK3588

## 🚀 What does it do?

dt-sim provides a complete Device Tree toolchain workflow:

1. **Compile DTS → DTB/DTBO** - Process Device Tree Source files with full include support
2. **Merge Base + Overlays** - Combine base DTB with multiple overlay DTBOs  
3. **Validate Everything** - Comprehensive validation with detailed error reporting
4. **Debug Issues** - Transparent process makes debugging trivial

## 🛠 Commands

| Command | Purpose | Example |
|---------|---------|---------|
| `dtc` | Compile DTS to DTB/DTBO text | `python dt-sim.py dtc base.dts -o base.dtb.txt` |
| `fdtoverlay` | Merge base + overlays | `python dt-sim.py fdtoverlay base.dtb.txt overlay.dtbo.txt -o final.dtb.txt` |

## ⚡ Quick Start

```bash
# 1. Compile base device tree
python dt-sim.py dtc rk3588-board.dts -o output/base.dtb.txt --validate -v

# 2. Compile overlay device trees
python dt-sim.py dtc enable-wifi.dts -o output/wifi.dtbo.txt --validate -v
python dt-sim.py dtc enable-display.dts -o output/display.dtbo.txt --validate -v

# 3. Merge everything into final DTB
python dt-sim.py fdtoverlay output/base.dtb.txt output/wifi.dtbo.txt output/display.dtbo.txt -o output/final.dtb.txt --verbose --show-changes
```

## 📝 Example Usage

### Complete Workflow
```bash
# Real-world example with RK3588 SoC
cd dt-sim

# Compile complex SoC base (108 nodes, 72 phandles)
python dt-sim.py dtc input/complex/rk3588-board.dts -o output/complex/rk3588-board.dtb.txt --validate -v -I input/include/

# Compile WiFi overlay
python dt-sim.py dtc input/overlays/enable-wifi.dts -o output/overlays/enable-wifi.dtbo.txt --validate -v -I input/include/

# Compile display overlay  
python dt-sim.py dtc input/overlays/enable-display.dts -o output/overlays/enable-display.dtbo.txt --validate -v -I input/include/

# Merge all overlays
python dt-sim.py fdtoverlay output/complex/rk3588-board.dtb.txt output/overlays/enable-wifi.dtbo.txt output/overlays/enable-display.dtbo.txt -o output/final/complete-system.dtb.txt --verbose --show-changes
```

### Node Reference Overlays
```bash
# Test node reference functionality
python dt-sim.py dtc test_references.dts -o output/test.dtb.txt --validate --test-overlays -v
```

## 📄 Output Format

dt-sim generates comprehensive, human-readable DTB text files:

```
/*
 * Generated by dt-sim dtc - mimics compiled DTB
 * Source file: input/complex/rk3588-board.dts
 * Total nodes: 108
 * Total phandles: 72
 */

/dts-v1/;

/*
 * ===== PHANDLE TABLE =====
 * phandle_map: {
 *   cpu0: 0x01 (/cpus/cpu@0)
 *   gic: 0x0c (/interrupt-controller@fe600000)
 *   gpio1: 0x35 (/pinctrl/gpio@fec20000)
 * }
 */

/ {
    compatible = "rockchip,rk3588-evb", "rockchip,rk3588"; /* rk3588-board.dts:2 */
    model = "Rockchip RK3588 Development Board"; /* rk3588-board.dts:2 */
    
    cpus {
        cpu0: cpu@0 {
            /* phandle: 0x01 (auto-assigned) */
            compatible = "arm,cortex-a55"; /* rk3588-soc.dtsi:45 */
            device_type = "cpu"; /* rk3588-soc.dtsi:45 */
        };
    };
};
```
